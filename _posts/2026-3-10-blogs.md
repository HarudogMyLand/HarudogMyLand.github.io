作为习惯了循规蹈矩的人，我一直自认为计算机不过是按部就班一步步执行的流水线。直到今晚，我手贱点开了Docker Desktop的安装包，命运的齿轮开始咔咔作响——不是朝着美好的方向，而是直接碾过了我的C盘。下载完成的那一刻，系统弹出了“空间不足”的提示，我还没来得及反应，整个电脑就像被人按住了喉咙：Ubuntu卡死，bash卡死，文件资源管理器卡死，连任务管理器都卡在半路摆烂。整个系统陷入了一种诡异的“I/O窒息”状态，硬盘似在哭泣。

就在我点击了安装包的那一刻，Docker Desktop在Windows上按照默认用WSL2引擎的指示，把所有镜像和容器塞进一个叫`ext4.vhdx`的虚拟硬盘文件里，藏在我根本想不到的路径深处：

```
C:\Users\Me\AppData\Local\Docker\wsl\data\
```

这玩意儿有个致命特性——只进不出。它像个貔貅，吃进去就不吐出来，眼睁睁看着C盘从黄色变成红色，最后归零。字面意义上的。

在C盘为0字节的环境下，任何命令都是奢望，连docker prune都报500错误。我只能物理降压——手动摸到C盘，看见什么没用的安装包、临时文件就删，像在废墟里刨食物，勉强腾出1GB喘口气。接着是暴力拆解：

结果`wsl --shutdown`卡死。

因此只能试试

```
taskkill /f /im wslservice.exe /t
taskkill /f /im vmmem.exe /t
```

再补一个`net stop LxssManager`。一套组合拳下来，系统终于松开了咬紧的牙关。

活过来之后，我下定决心：必须把Docker从C盘这个火坑里迁出去。过程像给系统做器官移植——先导出镜像备份：

```
wsl --export docker-desktop D:\DockerWSL\docker-engine.tar
```

然后注销C盘的原版：

```
wsl --unregister docker-desktop
```
最后导入到D盘重新安家：

```
wsl --import docker-desktop D:\DockerWSL\engine D:\DockerWSL\docker-engine.tar --version 2
```

中间还被“服务已存在”卡了一下，发现是Docker后台服务在搞鬼，去服务管理器里暂时停掉Docker Desktop Service，世界清静了。

如果搬起石头砸自己的脚，最好的回击就是看透它的逻辑，然后驯服它……或者下一次继续被他骑着:(
