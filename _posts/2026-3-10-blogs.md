2026年3月10日，为了水课科研课堂项目我不得不下载Docker Desktop来辅助学习，但是却因为Docker下载过程中磁盘空间问题凿沉了巨大困扰。

当然，此事必然有我自己的责任，由于懒惰，许多软件及其生成的垃圾文件全部堆积在C盘，使得下载Docker Desktop时C盘空间仅剩40GB，这当然不是一个吉利的数字。双击打开Docker Desktop安装包后，系统直接弹出「磁盘空间不足」提示。还没来得及点取消，整台电脑直接瘫痪。

Ubuntu终端、Git Bash和文件资源管理器卡住不动，任务管理器加载不出来，磁盘被占满，读写瘫痪，跟窒息了一样。

默认情况下，Docker Desktop会启用WSL2引擎，所有的镜像、容器数据，全部存在一个名为 `ext4.vhdx` 的虚拟磁盘文件里。这个文件默认藏在C盘的用户目录深处，普通人根本不会注意到：

```Plain
C:\Users\Me\AppData\Local\Docker\wsl\data\
```

而且这个虚拟盘文件有个**致命缺点**：只会膨胀、不会自动收缩。哪怕删除Docker镜像、清空容器，vhdx文件的占用空间也不会变小。

C盘彻底爆满之后，所有常规修复手段全部失效。

清理Docker垃圾的 `docker system prune` 命令报500错误，完全用不了。系统没有任何多余空间运行缓存、临时文件，所有程序全部锁死。

我手动进入C盘目录，挨个删除无用的安装包、缓存垃圾、临时文件，挤出1GB可用空间，让电脑勉强恢复呼吸，能正常执行命令。

有了一点空间后，我第一时间尝试关闭WSL服务来解除卡死状态，先执行常规关机命令：

```Plain
wsl --shutdown
```

又是直接卡死，毫无反应。

常规手段无效，只能上强制结束进程的组合命令，干掉WSL核心进程：

```Plain
taskkill /f /im wslservice.exe /t
taskkill /f /im vmmem.exe /t
```

再停止WSL核心服务，彻底终止占用：

```Plain
net stop LxssManager
```

卡死的系统终于彻底解脱，恢复正常运行。

**绝对不能再让Docker待在C盘**。全程实操命令如下：

**1. 先备份Docker原始镜像与容器数据**

将C盘默认的Docker WSL环境完整导出为备份文件，存到D盘目录：

```Plain
wsl --export docker-desktop D:\DockerWSL\docker-engine.tar
```

**2. 注销C盘原始的Docker WSL实例**

备份完成后，卸载C盘原本的Docker WSL环境，释放C盘空间：

```Plain
wsl --unregister docker-desktop
```

**3. 导入备份数据，部署到D盘**

把刚刚导出的备份文件，重新挂载导入到D盘指定目录，完成迁移：

```Plain
wsl --import docker-desktop D:\DockerWSL\engine D:\DockerWSL\docker-engine.tar --version 2
```

迁移过程中我还遇到了「服务已存在」的报错，排查后发现是Docker后台常驻服务没有关闭，占用进程导致导入失败。打开系统服务管理器，手动临时关闭**Docker Desktop Service**，再重新执行导入命令，成功。

Docker本身没问题，WSL2也没问题，坏就坏在默认C盘存储+虚拟盘只涨不缩的机制，悄无声息毁灭系统盘。
