这篇文章记录两件事：常用 Markdown 语法，以及如何在 academicpages 这类 Jekyll 站点里发布一篇博客。

## Markdown 基础语法

### 标题

Markdown 使用 `#` 表示标题层级：

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
```

博客正文里建议从 `##` 开始使用小节标题，因为文章标题已经是页面的 `h1`。

### 段落与换行

普通文字直接书写即可。段落之间空一行：

```markdown
这是第一段。

这是第二段。
```

如果只是想强制换行，可以在行尾添加两个空格，但一般不推荐频繁使用。

### 加粗、斜体、删除线

```markdown
**加粗**
*斜体*
~~删除线~~
```

效果：

**加粗**

*斜体*

~~删除线~~

### 列表

无序列表：

```markdown
- 第一项
- 第二项
- 第三项
```

有序列表：

```markdown
1. 第一步
2. 第二步
3. 第三步
```

### 链接

```markdown
[AcademicPages](https://academicpages.github.io/)
```

如果链接到站内页面，优先使用相对路径或 Jekyll 生成后的页面地址。

### 图片

```markdown
![图片说明](/images/example.png)
```

图片通常放在 `images/` 或自己约定的资源目录中。

### 引用

```markdown
> 这是一段引用。
> 可以用来摘录文献、提示重点或保留一段原话。
```

效果：

> 这是一段引用。
> 可以用来摘录文献、提示重点或保留一段原话。

### 行内代码

行内代码用反引号包裹：

```markdown
运行 `bundle exec jekyll serve` 启动本地预览。
```

效果：运行 `bundle exec jekyll serve` 启动本地预览。

### 代码块

多行代码使用三个反引号：

````markdown
```bash
bundle exec jekyll serve
```
````

效果：

```bash
bundle exec jekyll serve
```

### 表格

```markdown
| 字段 | 作用 |
| --- | --- |
| title | 文章标题 |
| date | 发布时间 |
| tags | 标签 |
| columns | 专栏归属 |
```

效果：

| 字段 | 作用 |
| --- | --- |
| title | 文章标题 |
| date | 发布时间 |
| tags | 标签 |
| columns | 专栏归属 |

### 分隔线

```markdown
---
```

注意：文章最顶部的 `---` 是 YAML Front Matter 的边界，不是正文分隔线。正文中使用分隔线时，前后最好留空行。

## AcademicPages 发博客流程

academicpages 基于 Jekyll。博客文章通常放在 `_posts/` 文件夹中。

### 第一步：创建文章文件

文件名格式建议为：

```text
YYYY-MM-DD-title.md
```

例如：

```text
2026-06-26-blog-columns.md
```

Jekyll 会根据文件名和 front matter 中的 `date` 生成归档顺序。

### 第二步：写 Front Matter

每篇文章顶部需要一段 YAML Front Matter：

```yaml
---
title: 'My First Blog Post'
date: 2026-06-26
excerpt: "A short description for archive pages."
tags:
  - Markdown
  - Blog
columns:
  - writing-lab
---
```

常用字段：

| 字段 | 是否常用 | 说明 |
| --- | --- | --- |
| `title` | 是 | 文章标题 |
| `date` | 是 | 发布时间 |
| `excerpt` | 是 | 归档页摘要 |
| `tags` | 是 | 标签筛选用 |
| `columns` | 可选 | 专栏归属 |
| `permalink` | 可选 | 自定义文章 URL |

### 第三步：撰写正文

Front Matter 结束后，就可以直接写 Markdown 正文：

```markdown
## Introduction

这里写文章内容。

## Notes

- 第一条笔记
- 第二条笔记
```

### 第四步：加入专栏

如果文章属于某个专栏，在 front matter 中添加：

```yaml
columns:
  - writing-lab
```

专栏本身定义在：

```text
_data/blog_columns.yml
```

例如：

```yaml
- key: writing-lab
  title: Writing Lab
  description: Blogging workflow, Markdown, site maintenance, and publishing experiments.
  icon: fas fa-pen-nib
```

文章里的 `columns` 写的是专栏的 `key`，不是标题。

### 第五步：本地预览

在项目根目录运行：

```bash
bundle exec jekyll serve
```

然后打开：

```text
http://localhost:4000/
```

如果只想检查博客归档页，可以访问：

```text
http://localhost:4000/year-archive/
```

### 第六步：检查页面

发布前建议检查：

- 标题层级是否从 `##` 开始。
- 代码块是否指定语言，例如 `bash`、`yaml`、`python`。
- 图片路径是否正确。
- tag 是否过多。
- `columns` 是否写成了正确的专栏 key。
- 本地预览是否能正常打开文章页面。

## 一个最小文章模板

```markdown
---
title: 'New Blog Post'
date: 2026-06-26
excerpt: "One sentence summary."
tags:
  - Blog
columns:
  - writing-lab
---

## Introduction

Write the opening here.

## Main Notes

- Point one.
- Point two.

## Conclusion

Wrap up the post here.
```

这就是在 academicpages 中发布博客的基本流程：创建 `_posts` 文件、写 front matter、写 Markdown 正文、本地预览，然后提交发布。
